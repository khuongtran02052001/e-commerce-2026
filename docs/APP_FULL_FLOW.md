## 1) High-Level Architecture

```mermaid
flowchart LR
  U[User Browser] --> FE[Next.js Frontend]
  FE --> NA[NextAuth Session/JWT]
  FE --> RC[restClient / data layer]
  RC --> BE[service-system REST API]
  BE --> DB[(Database)]
```

## 3) Auth + Guard Flow

```mermaid
sequenceDiagram
  participant Browser
  participant FE as Next.js
  participant Auth as NextAuth
  participant API as REST API

  Browser->>FE: Sign in with Google
  FE->>API: POST /auth/google-login (googleIdToken)
  API-->>FE: accessToken
  FE->>Auth: Save accessToken in session token
  Browser->>FE: Open protected page
  FE->>API: GET /auth/me (Bearer accessToken)
  API-->>FE: { isAdmin, isEmployee, employeeRole, ... }
  FE-->>Browser: Render by role/permissions
```

## 4) Core Business Flows

### 4.1 Customer Order Flow

```mermaid
flowchart LR
  C1[User creates order] --> C2[POST /orders]
  C2 --> C3[BE loads products by productId]
  C3 --> C4[Validate exists + stock]
  C4 --> C5[Compute subtotal/tax/shipping/total from env]
  C5 --> C6[Save order + order products]
  C6 --> C7[User tracks at /user/orders]
```

### 4.2 Employee Workflow Flow

```mermaid
flowchart LR
  O[Order] --> CC1[CALL_CENTER: address-confirm]
  CC1 --> CC2[CALL_CENTER: order-confirm]
  CC2 --> P1[PACKER: pack]
  P1 --> P2[PACKER: assign-deliveryman]
  P2 --> D1[DELIVERYMAN: start-delivery]
  D1 --> D2[DELIVERYMAN: deliver or reschedule or failed]
  D2 --> A1[ACCOUNTS: receive-payment]
  A1 --> Done[Completed]
```

### 4.3 Admin Management Flow

```mermaid
flowchart TD
  AM[Admin opens /admin/employees]
  AM --> ARole[Assign/Remove employee role]
  AM --> AStatus[Suspend/Activate employee]
  ARole --> LoginAgain[User re-login to refresh JWT role claims]
  LoginAgain --> EMP["/employee page shows correct actions"]
```

## 5) Data Ownership

- FE owns:
  - rendering, UX state, guard redirects
  - role-based action visibility
- BE owns:
  - source of truth for auth/permissions
  - order pricing/tax/shipping calculations
  - stock validation
  - workflow state transitions + audit logs
